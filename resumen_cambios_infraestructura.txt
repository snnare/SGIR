2. INFRAESTRUCTURA DE PRUEBAS (DOCKER)
   Se configuró un entorno multi-motor heterogéneo en 'bd/docker-compose.yml':
   - PostgreSQL 16: Base de datos maestra de SGIR.
   - MySQL 5.7: Servidor Legacy para pruebas de compatibilidad.
   - MySQL 8.0: Servidor moderno con seguridad mejorada.
   - Oracle 19c: Infraestructura de Misión Crítica (Esquema RH-UAEMEX).
   - MongoDB Latest: Almacenamiento NoSQL.
   - SSH Server: Emulación de servidor Linux para métricas de SO (Puerto 2222).

3. ESTRATEGIA DE MONITOREO SRE (GOLDEN SIGNALS)
   Se implementaron y validaron credenciales para monitoreo no invasivo:
   
   A. MySQL (5.7/8.0) - monitor_user / monitor_pass
      - Métricas: Uptime, Threads_connected/running, Slow_queries, Data/Index Size.
   
   B. Oracle 19c - monitor_user / monitor_pass (PDB: ORCLPDB1)
      - Métricas: Open Mode, Tablespace Usage (%), FRA Space, Invalid Objects (RH-UAEMEX).
   
   C. MongoDB - monitor_user / monitor_pass (Auth: admin)
      - Métricas: WiredTiger Tickets, OpCounters, Page Faults, Index Size.
      - ESTADO: Validado mediante 'mongosh' (Acceso a clusterMonitor y readAnyDatabase exitoso).
   
   D. Linux Server (SSH) - sgir_user / sgir_pass (Puerto 2222)
      - Métricas: Load Average, RAM Usage, Disk Space (%), i-nodes.
